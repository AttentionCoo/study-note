# 现代大模型自回归生成与 KV 缓存详解

> 本文深入剖析大语言模型（LLM）在推理（生成）阶段的核心机制，从自回归生成的最朴素实现出发，逐步解释 **KV 缓存** 的设计动机、数学原理、计算流程及其与 QKV 的关系，最后给出完整示例与工程实现要点。

---

## 1. 自回归生成的基本流程

大模型（如 GPT、Llama 等）采用**自回归**（autoregressive）方式生成文本：一次只预测并输出一个 token（词或子词），然后将新 token 拼接到输入序列末尾，再次送入模型以生成下一个 token，直至遇到结束符。

### 1.1 朴素实现（无优化）

每次生成新 token 时，将 **整个历史序列**（提示词 + 已生成的所有 token）重新输入 Transformer 的所有层，重新计算所有位置的隐状态和注意力。

- 假设当前序列长度为 $n$。
- 自注意力需要为每个位置计算与所有位置的关联，复杂度 $O(n^2)$。
- 总共生成 $N$ 个 token，全部累计复杂度 $O(N^3)$ 级别（若逐 token 生成，总计算量 $\sum_{n=1}^{N} n^2 \approx O(N^3)$）。

这种重复计算是巨大的浪费，因为连续两次生成之间，历史序列的绝大多数运算结果没有变化。

### 1.2 关键观察：自注意力中的不变量

在 Transformer 的自注意力层中，对于每一个已存在的 token：

- 其 **Key（K）** 和 **Value（V）** 只取决于 token 自身的向量表示和权重矩阵 $W_K, W_V$。
- 由于后续生成的 token 不会改变之前 token 的表示（单向注意力，因果掩码），因此历史 token 的 K 和 V 是固定不变的，可被复用。
- 而 Query（Q）是每次根据“当前正在处理的位置”动态生成的，不会用于其他位置的生成（除非那个位置成为新的“当前 token”）。

基于此，自然引出 **KV 缓存**。

---

## 2. KV 缓存：原理与作用

### 2.1 定义

**KV 缓存** 是指在推理过程中，将每一层自注意力中**已经计算过的 token 的 Key 和 Value 向量**保存在显存中，供后续 token 的生成步骤直接读取，避免重复计算。

### 2.2 与 Q、K、V 的关系

- 自注意力核心公式（单头简化）：

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V
$$

- 对于序列中的 **当前 token**（例如第 $t$ 个位置），其 Query 为 $q_t$，需要与所有位置的 Key（$k_1, \dots, k_t$）计算点积，得到注意力权重，再与所有 Value（$v_1, \dots, v_t$）加权求和。
- **KV 缓存** 在生成第 $t+1$ 个 token 时，已存有前 $t$ 个 token 的 $k_{1:t}$ 和 $v_{1:t}$。
- 第 $t+1$ 个 token 只需根据其本身计算 $q_{t+1}$、$k_{t+1}$、$v_{t+1}$，然后 $q_{t+1}$ 与缓存中的 $k_{1:t+1}$（包含新 token 的 k）计算注意力，再与新缓存的 $v_{1:t+1}$ 加权求和。
- 仅 $k_{t+1}$、$v_{t+1}$ 被追加到缓存，历史 Q 全部丢弃。

### 2.3 效果对比

| 实现方式   | 单步计算量         | 显存占用                             | 备注               |
| ---------- | ------------------ | ------------------------------------ | ------------------ |
| 无缓存     | $O(t^2)$           | $O(t)$ 临时的激活值                  | 需要重算历史，极慢 |
| 有 KV 缓存 | $O(t)$（线性增长） | 每层缓存大小 $2 \times t \times d_k$ | 标准生成方法       |

## 3. Q、K、V 的计算细节

### 3.1 从 Token 到向量

每个 token 经嵌入层加上位置编码后得到向量 $x \in \mathbb{R}^{d_\text{model}}$。

### 3.2 三个线性变换

对输入 $x$ 分别乘以三个独立的权重矩阵（无偏置通常是常见做法）：

$$
\begin{aligned}
q &= x \cdot W_Q \quad (W_Q \in \mathbb{R}^{d_\text{model} \times d_k}) \\
k &= x \cdot W_K \quad (W_K \in \mathbb{R}^{d_\text{model} \times d_k}) \\
v &= x \cdot W_V \quad (W_V \in \mathbb{R}^{d_\text{model} \times d_v})
\end{aligned}
$$

- 在多头注意力中，$d_k = d_v = d_\text{model} / h$，$h$ 为头数。例如 GPT-2 小模型：$d_\text{model}=768, h=12, d_k=64$。
- 每个头有各自的 $W_Q^{(i)}, W_K^{(i)}, W_V^{(i)}$，最终结果拼接后投影。

### 3.3 矩阵批量计算

当一次性处理长度为 $L$ 的完整序列时，输入矩阵 $X \in \mathbb{R}^{L \times d_\text{model}}$：

$$
Q = X W_Q \quad (L \times d_k), \quad K = X W_K, \quad V = X W_V
$$

### 3.4 逐 Token 生成时的应用

- **预填充阶段**：将提示词（prompt）一次性送入模型，计算所有 token 的 K、V 并存入缓存。
- **生成阶段**：
  1. 取出序列最后一个 token（或最新生成 token）的嵌入向量 $x_t$。
  2. 只计算 $q_t, k_t, v_t$。
  3. 从缓存中取出所有历史的 K（$K_\text{cache}$）和 V（$V_\text{cache}$），并加入 $k_t, v_t$。
  4. 计算注意力输出：$\text{softmax}\left(\frac{q_t K_\text{all}^T}{\sqrt{d_k}}\right) V_\text{all}$。
  5. 将 $k_t, v_t$ 存入缓存，丢弃 $q_t$。
  6. 前向传播继续经过层归一化、FFN 等，最终输出下一个 token 的概率。

## 4. 注意力机制中的加权求和详解

### 4.1 步骤分解（通过例子）

假设已经生成 token A、B、C，正在生成 token D。

**缓存**：
$$
K_\text{cache} = [k_A, k_B, k_C], \quad V_\text{cache} = [v_A, v_B, v_C]
$$
**新计算**（token D 的向量 $x_D$）：
$$
q_D, k_D, v_D
$$

**计算注意力权重**：

1. 将 $k_D$ 加入缓存（自注意力中，当前 token 也要看到自己），形成完整 K 集合 $K = [k_A, k_B, k_C, k_D]$。
2. 计算分数（缩放点积）：
   $$
   s_i = \frac{q_D \cdot k_i}{\sqrt{d_k}} \quad i \in \{A,B,C,D\}
   $$
3. Softmax 归一化：
   $$
   w_i = \frac{e^{s_i}}{\sum_j e^{s_j}}
   $$
   权重 $w_i$ 表示 token D 对 token i 的关注程度，且 $\sum w_i = 1$。

**加权求和**：
$$
\text{output}_D = w_A v_A + w_B v_B + w_C v_C + w_D v_D
$$

- 这里每个 $v_i$ 是 Value 向量，携带 token i 的“信息内容”。
- 加权求和本质上是 **依据相关性权重从历史信息中抽取并混合**。

### 4.2 为什么需要加权求和？

1. **信息聚合**：自然语言理解需要上下文，当前预测必须综合前面所有词的信息，而非仅看最后一个词。例如在“他买了一本书，他非常喜欢___”中，填充“它”时需要高权重关注“书”的 Value。
2. **动态选择**：模型通过 Q 和 K 的点积学出哪些位置的重要程度，实现了长距离依赖的动态捕获，不像 CNN/RNN 有固定的感受野或隐藏状态瓶颈。
3. **可变长度编码**：将不定长的历史信息压缩为固定维度的向量，供后续层和最终输出层使用。

---

## 5. 多头注意力中的 KV 缓存

现代 Transformer 均使用**多头注意力**。具体做法：

- 将 Q、K、V 拆分为 $h$ 个头，每个头独立计算注意力。
- 在 KV 缓存中，每个头需独立存储自己的 K 和 V。即每层缓存维度为 $2 \times h \times t \times d_k$。
- 生成时，每个头只计算新 token 在该头的 $q, k, v$，然后和自己的缓存完成注意力。

---

## 6. 完整示例：带 KV 缓存的生成流程

**场景**：提示词 “今天天气” （三个 token），需要生成后续 token。

### 步骤 0（预填充）

- 输入 [今天, 天气, <eos?>]，模型计算所有层的 Q、K、V，将 K、V 存入每层的缓存。
- 拿到最后位置（“气”）的输出，预测下一个 token，得到 “很”。

### 步骤 1（生成第一个新 token “很”）

- 只将 token “很” 的嵌入向量 $x_{\text{很}}$ 送入模型。
- 每层自注意力：
  - 计算 $q_{\text{很}}, k_{\text{很}}, v_{\text{很}}$。
  - 从缓存加载之前三个 token 的 K 和 V，加上 $k_{\text{很}}, v_{\text{很}}$。
  - 用 $q_{\text{很}}$ 与所有 K 计算注意力权重，对 V 加权求和。
  - $k_{\text{很}}, v_{\text{很}}$ 存入缓存。
- 输出层给出下一个 token 概率，采样得到 “好”。

### 步骤 2（生成 “好”）

- 流程同上，输入 token “好”，加入对应 K、V 到缓存，预测出下一个 token ……

**缓存状态变化示意**：

| 步骤 | 输入 token                   | 缓存中存储的 K、V 对应的 token |
| ---- | ---------------------------- | ------------------------------ |
| 0    | (批量) “今天”、“天气”、“<s>” | “今天”、“天气”、“<s>”          |
| 1    | “很”                         | “今天”、“天气”、“<s>”、“很”    |
| 2    | “好”                         | 上述 + “好”                    |
| 3    | …                            | …持续追加                      |

## 7. 工程实现注意事项

- **预分配缓存**：由于最终长度未知，通常预分配一个最大长度的张量（如 `batch_size × num_layers × max_seq_len × num_heads × head_dim`），逐步填充。
- **因果掩码**：在自注意力中必须使用 mask 避免看到未来信息，但因生成时本身就是单向的，且 KV 缓存只包含历史，直接点积即可，无需额外掩码（注意预填充阶段需下三角 mask）。
- **int8/量化缓存**：为节省显存，可将缓存的 K、V 进行量化，例如 `LLM.int8()` 或 SmoothQuant 等方法。
- **Dynamic Batching / Continuous Batching**：为提升吞吐，不同请求的序列长度可能不同，KV 缓存管理成为复杂问题（如 vLLM 中的 PagedAttention）。

---

## 8. 总结

- **自回归生成** 是 LLM 的基础范式，每次产生一个 token 并拼接回输入。
- **朴素实现** 会导致重复计算，资源浪费巨大。
- **KV 缓存** 利用自注意力中历史 K、V 不变的性质，复用计算结果，将复杂度从 $O(n^2)$ 降到 $O(n)$（单步），为所有生产级大模型的核心技术。
- **Q、K、V** 通过输入向量与三个可训练权重矩阵相乘得到，其中 Q 用于查询，K 用于匹配，V 用于加权信息提取。
- **注意力加权求和**：当前 token 的 Q 与所有缓存的 K 进行点积得到相关性分数，经 softmax 转为权重，对相应的 V 进行加权求和，得到融合历史上下文的输出。该机制赋予模型动态、有选择地聚合上下文的能力。
- **完整流程**：预填充 → 逐 token 计算 Q、K、V → 更新缓存 → 输出下一个 token。

KV 缓存是 LLM 推理加速的基石，理解其工作原理对后续推理框架优化、长上下文方法（如 ALiBi、RoPE、PagedAttention）和推理硬件选型均有重要意义。

# 9. 理解

## 9.1

是的，你的理解基本正确，但现代大模型通常会做一个关键优化：**KV 缓存**。

- **朴素做法**（你描述的方式）：每次生成一个新 token，都把 **提示词 + 已生成的所有 token** 重新输入 Transformer 计算一遍。这会导致计算量随序列长度平方增长，非常低效。
- **实际做法**：使用 **KV 缓存**。因为自注意力中，已生成 token 的 Key 和 Value 在生成后续 token 时**不会改变**。模型会缓存这些值，每次只计算新 token 的 Key/Value 和注意力，把计算量从 $O(n^2)$ 降到 $O(n)$。

所以流程是：**每次输入上一个 token**（加上缓存的历史 KV） → Transformer 计算 → 生成下一个 token → 更新缓存。这就实现了高效的**自回归生成**。

另外，有一种**非自回归生成**会并行生成多个 token，但这并非主流做法，主要用在需要低延迟或特定任务的场景中。

## 9.2

**KV缓存**正是你在理解自回归生成时提到的 QKV 机制的直接优化产物。

### 它与 QKV 的关系

在 Transformer 的**自注意力层**，每个 token 都会生成三个向量：**Q**（Query）、**K**（Key）、**V**（Value）。  
计算当前 token 的注意力输出时，需要用它的 **Q** 去和所有已存在 token 的 **K** 做匹配（得到权重），再与对应的 **V** 加权求和。

关键观察：  

- 生成新 token 时，之前所有 token 的 **K** 和 **V** 已经计算过，并且**不会因为新 token 的到来而改变**（因为注意力只关注历史，不改变历史表征）。  
- 而新 token 自己的 **Q** 必须重新计算（因为它是根据当前上下文生成的），它的 **K** 和 **V** 也会被计算出来，并加入缓存供后续 token 使用。

**KV 缓存**就是：  

> 在每层自注意力中，将已经生成 token 的 **K** 和 **V** 保存下来。生成下一个 token 时，只计算新 token 的 Q、K、V，再用新 Q 与缓存中的所有 K 计算注意力，并用缓存中的 V 完成加权求和。

### 效果对比

- **无缓存**：每次生成 token，所有历史 token 的 Q、K、V 都要重新算 → 计算量 $O(n^2)$ 且无法随长度增长。
- **有 KV 缓存**：只算新 token 的 Q、K、V（$O(1)$ 长度相关计算），注意力部分依然是 $O(n)$（因为要跟缓存里所有 K 做点积），但避免了平方级的重算。

所以，**KV 缓存本质上是“记住历史的 K 和 V，遗忘历史的 Q”**——因为历史的 Q 在生成后续 token 时不会被用到（注意力总是单向的，只关心当前 Q 与过去所有 K、V 的关系）。

## 9.3

QKV 的计算本质上是**三个不同的线性变换**。我们来分步拆解，假设你有一个已经向量化的输入 token。

### 1. 输入：一个 token 的向量

假设模型维度是 `d_model`（比如 512）。每个 token（不论位置）都会先被表示为一个 **d_model 维的向量**。

### 2. 核心操作：乘以三个不同的权重矩阵

这个向量会被送入三个全连接层（线性层，没有偏置是常见做法），分别对应 Q、K、V：

- **计算 Q**：向量 × **W_Q** 权重矩阵 → 得到 Query 向量（维度通常为 `d_k`）
- **计算 K**：向量 × **W_K** 权重矩阵 → 得到 Key 向量（维度 `d_k`）
- **计算 V**：向量 × **W_V** 权重矩阵 → 得到 Value 向量（维度 `d_v`）

**关键点**：W_Q、W_K、W_V 是**独立训练**的不同参数矩阵，但通常 `d_k = d_v` 且等于 `d_model / 头数`（多头注意力会拆分）。

### 3. 实际实现：矩阵批处理

虽然概念上逐个 token 计算，但代码里为了效率，通常是**整个序列的矩阵一次算完**：

假设输入矩阵 X 形状为 `[序列长度, d_model]`，那么：

- **Q = X · W_Q** （形状 `[L, d_k]`）
- **K = X · W_K** （形状 `[L, d_k]`）
- **V = X · W_V** （形状 `[L, d_v]`）

这样一次前向传播就得到了所有 token 的 Q、K、V。

### 4. 在逐 token 生成中的角色

结合你之前问的 **KV 缓存**：

- **新 token**：需要给它完整计算 Q、K、V（因为它刚进来）。
- **历史 token**：它们的 K 和 V 已经缓存了，不需要用上面的矩阵再算一遍。但注意，理论上缓存的正是 `[历史token] × W_K` 和 `[历史token] × W_V` 的结果。
- **历史 token 的 Q**：根本不需要缓存，因为后续生成不再使用旧的 Q（每次只拿**当前新 token 的 Q** 去和历史 K 做点积）。

### 总结一句话

> Q、K、V 分别是输入向量与三个不同训练好的权重矩阵 **W_Q、W_K、W_V** 做矩阵乘法得到的。其中 **K 和 V** 会被逐 token 地缓存起来用于后续生成，而 **Q** 每次都只针对当前生成的新 token 即时计算。

## 9.4

这句话描述了自注意力机制中的核心操作，我用一个具体例子来拆解。

### 先明确几个符号

假设我们已经生成了 token A、B、C（历史），正在生成新 token D（当前）。

- 缓存里有：A、B、C 的 **K** 向量（$K_A, K_B, K_C$）和 **V** 向量（$V_A, V_B, V_C$）。
- 刚刚计算出 D 的 **Q** 向量（$Q_D$）、**K** 向量（$K_D$）、**V** 向量（$V_D$）。  
  注意：D 的 K 和 V 也会**加入缓存**，供后续 token E 使用；但在计算 D 自身的注意力时，是否包含 D 自己的 K、V？通常包含，所以缓存里实际还应有 $K_D, V_D$，但为了简单，我先用历史做例子，最后补充。

---

### 第一步：用 Q 与所有 K 计算注意力权重

计算 **注意力分数**：  
$ \text{score}_i = Q_D \cdot K_i $ （点积，通常会除以 $\sqrt{d_k}$ 做缩放）

得到分数向量：$[s_A, s_B, s_C]$ （也可能包含 $s_D$）

然后对这些分数做 **Softmax**，得到权重：  
$ w_i = \frac{e^{s_i}}{\sum_j e^{s_j}} $  
并且所有权重之和为 1。

所以现在有 $w_A, w_B, w_C$，表示当前 token D 应该**分别关注**历史 token A、B、C 的程度。

---

### 第二步：用这些权重对 V 做加权求和

**加权求和公式**：  
$$
\text{Output}_D = w_A \cdot V_A + w_B \cdot V_B + w_C \cdot V_C
$$

含义：  

- **V 向量**（Value）代表每个 token 所携带的“信息内容”。
- **权重** $w$ 代表该信息对当前 token 的重要性。
- 加权求和就是把所有历史信息按重要性混合起来，得到一个**聚合向量**。

这个聚合向量 $\text{Output}_D$ 就是 D 的**自注意力层输出**。它随后会经过残差连接、层归一化和前馈网络，最终用来预测下一个 token E。

---

### 为什么要完成加权求和？

一句话回答：**为了从历史中提取最相关的信息，并压缩成当前 token 可用的表示。**

具体两点原因：

1. **信息聚合**  
   语言是有上下文的。预测下一个词时，需要参考前面多个词，而不是只看最后一个词。加权求和让模型能够**有选择地集中注意力**：比如在 “她很喜欢吃苹果，她每天都会买一个___” 中，要预测 “苹果”，模型会给 “苹果” 的 V 很高的权重，而给 “喜欢” 较低的权重。

2. **可变长度上下文**  
   没有加权求和，而是仅仅拼接或简单平均，都会丢失位置和相关性信息。加权求和通过训练出的 Q、K 矩阵，让模型**动态决定**从多远的历史、哪些词中抽取信息。

### 关于当前 token 自己的 K 和 V

在实际代码中（如 GPT），生成新 token D 时，也会**把 D 自己的 K_D 和 V_D 先加入缓存**，然后让 $Q_D$ 与包括 $K_D$ 在内的所有 K 计算权重，从而 D 也能关注到自己（这称为 **自注意力包含自身位置**）。  
此时加权和会多出一项 $w_D \cdot V_D$，这样当前 token 的输出也会融合自身的原始信息。

但对你问题里的理解来说，核心逻辑是一样的：**Q 与 K 得到相关性权重，权重与 V 相乘并求和，得到融合后的输出。
