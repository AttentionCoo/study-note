# GPT 与 Transformer 架构详解：从核心区别到完整运行流程

## 1. 核心结论速览

| 模型 | 架构类型 | 注意力机制 | 典型用途 |
|------|----------|------------|----------|
| **GPT** | Decoder‑only（仅解码器） | 因果（单向）自注意力 | 语言建模、开放域文本生成 |
| **原始 Transformer** | Encoder‑Decoder（编码器-解码器） | Encoder 双向自注意力 + Decoder 因果自注意力 & 交叉注意力 | 机器翻译、摘要等 Seq2Seq 任务 |

> 一句话区分：**GPT 是“单塔”仅解码器结构，通过掩码自注意力只看向左侧上文；原始 Transformer 是“双塔”结构，编码器双向理解源序列，解码器交叉关注源序列并自回归生成目标序列。**

---

## 2. GPT（Decoder‑only）完整运行流程

**场景**：自回归生成 `"今天天气"` → 预测下一个词。

### 2.1 输入处理
1. **Tokenization & Embedding**  
   输入序列 `["今天", "天气"]` 转换为 token id，再通过嵌入层 `W_e` 映射为向量，形状 `[seq_len=2, d_model]`。

2. **位置编码**  
   加入可学习的位置嵌入（或正弦位置编码），使模型感知顺序。

最终得到输入表示 `X ∈ ℝ²ˣᵈᵐᵒᵈᵉˡ`。

### 2.2 堆叠 Decoder 层（L 层，如 GPT‑3 有 96 层）
每一层包含两个子层（均带有残差连接和层归一化）：

#### a) 带因果掩码的多头自注意力（Masked Multi‑Head Self‑Attention）
- 由 `X` 线性投影得到 `Q, K, V`，多头拆分。
- 计算注意力分数：`Scores = QKᵀ / √d_k`，形状 `[2, 2]`。
- **因果掩码**：将 `i > j`（未来位置）的分数置为 `-∞`，确保第 `i` 个 token 只能关注第 `0...i` 个 token。
- Softmax 归一化 → 加权聚合 `V` → 拼接多头输出。
- 残差：`Out₁ = LayerNorm(X + MultiHead(Q,K,V))`。

#### b) 前馈网络（FFN）
- 通常为两层线性 + 激活函数（GELU 或 ReLU）：
  `FFN(x) = GELU(xW₁ + b₁)W₂ + b₂`，中间维度 `4*d_model`。
- 残差：`Out₂ = LayerNorm(Out₁ + FFN(Out₁))`。

重复 L 层后，得到最终隐状态 `H ∈ ℝ²ˣᵈᵐᵒᵈᵉˡ`。

### 2.3 输出与自回归生成
1. 取**最后一个位置**的隐状态 `h = H[-1, :]`（生成时仅需最后一步的表示）。
2. 通过输出词表映射（Unembedding）得到 logits：`logits = h · W_outᵀ`（形状 `[vocab_size]`）。
3. Softmax 得出下一个 token 的概率分布，采样或取 argmax 得到预测词，例如 `"好"`。
4. 将预测词追加到输入序列（`["今天","天气","好"]`），重复上述流程直至生成结束符 `<eos>`。

**本质**：每一步条件概率 `P(x_t | x_{<t})`，仅依赖历史信息，不涉及外部编码。

## 3. 原始 Transformer（Encoder‑Decoder）完整运行流程

**场景**：英语→德语翻译，源句 `"I am a student"`，目标句 `"Ich bin ein Student"`。

### 3.1 Encoder 端（双向理解源语言）
1. **输入嵌入 + 位置编码**：与 GPT 类似，得到 `X_enc ∈ ℝˢʳᶜ_ˡᵉⁿ × ᵈᵐᵒᵈᵉˡ`。
2. **N 个相同的 Encoder 层**（原论文为 6 层），每层包含：
   - **多头自注意力（无掩码）**：每个 token 可以关注所有其他 token（双向），充分融合上下文。
   - **FFN**。
   - 每个子层后：残差连接 + 层归一化。
3. 输出最终的 **编码器记忆 `Mem`**，形状 `[src_len, d_model]`，它包含了整句的深层双向表示。

### 3.2 Decoder 端（条件自回归生成）
**训练时**（并行计算）：
- 输入目标序列 `["<sos>", "Ich", "bin", "ein", "Student"]`，经嵌入 + 位置编码得到 `X_dec`。
- 堆叠 N 层 Decoder，每层包含 **三个** 子层：

  **① 带因果掩码的自注意力（Masked Self‑Attention）**  
  - 与 GPT 相同，只能看到当前位置及以前的 token。

  **② 交叉注意力（Encoder‑Decoder Attention）**  
  - `Q` 来自上一层 Decoder 的输出，`K, V` 来自 Encoder 的 `Mem`。
  - 每条目标位置都会查询所有源语言位置，计算关联分数（无掩码）。
  - 这使模型学会“对齐”：生成当前目标词时该关注源句的哪些词。

  **③ 前馈网络（FFN）**

  所有子层同样配备残差 + 层归一化。

- 最终 Decoder 输出 `H_dec`，通过线性层 + Softmax 得到每个位置的预测分布，与真实标签计算交叉熵损失。

**推理时**（自回归）：
- 初始输入仅为 `<sos>`。
- 每一步：走完整个 Decoder（自注意力仅看已生成部分，交叉注意力始终使用同一个 `Mem`），输出最后一个位置的概率，采样得到新词。
- 将新词接在输入后面，重复直到 `<eos>`。

## 4. Transformer Decoder 具体计算的是什么？如何计算？

**计算目标**：给定 **Encoder 输出的源语言记忆 `Mem`** 和 **已生成的部分目标序列 `y_{<t}`**，计算下一个目标词的条件概率分布 `P(y_t | y_{<t}, x)`。

### 计算步骤（以一层 Decoder 为例，原始 Transformer 有 N 层）

假设当前输入目标序列表示为 `x ∈ ℝᵗˣᵈᵐᵒᵈᵉˡ`（第一层为词嵌入+位置编码），`Mem ∈ ℝˢʳᶜˡᵉⁿˣᵈᵐᵒᵈᵉˡ`。

#### 子层 1：掩码多头自注意力
- 从 `x` 投影出 `Q, K, V`（多头）。
- 分数矩阵 `QKᵀ` 上应用**下三角掩码**（未来位置置 `-∞`）。
- 加权求和 `V`，得到上下文表示 `z₁`。
- 输出：`x₁ = LayerNorm(x + z₁)`。

#### 子层 2：Encoder‑Decoder 交叉注意力
- `Q` 来自 `x₁`，`K, V` 来自 `Mem`（固定不变）。
- 计算 `Scores = Q_dec · K_encᵀ / √d_k`，形状 `[t, src_len]`。
- **无掩码**，可看到全部源信息。
- Softmax 加权 `V_enc` 得到 `z₂`。
- 输出：`x₂ = LayerNorm(x₁ + z₂)`。

#### 子层 3：前馈网络
- `FFN(x₂) = max(0, x₂W₁ + b₁)W₂ + b₂`（或 GELU）。
- 输出：`x₃ = LayerNorm(x₂ + FFN(x₂))`。

重复 N 层后，得到最终隐状态 `H ∈ ℝᵗˣᵈᵐᵒᵈᵉˡ`。  
最后映射：`logits = H · W_outᵀ`（`W_out` 常与输入词嵌入共享），形状 `[t, vocab_size]`，Softmax 后即得到每个位置预测的下一个词概率。

## 5. GPT 与原始 Transformer 详细对比

| 特性 | 原始 Transformer | GPT (Decoder‑only) |
|------|------------------|---------------------|
| **架构** | Encoder‑Decoder（双塔） | Decoder‑only（单塔） |
| **注意力掩码** | Encoder：无（双向）<br>Decoder：因果掩码（单向） | 全部因果掩码（单向） |
| **交叉注意力** | ✅ Decoder 中包含，用于关注源信息 | ❌ 不存在，完全依赖自注意力 |
| **输入输出** | 源序列 + 目标序列 | 单一文本前缀 |
| **典型任务** | 机器翻译、摘要、Seq2Seq | 语言建模、开放式文本生成 |
| **训练目标** | 监督交叉熵（预测目标序列） | 自回归语言建模（预测下一个 token） |
| **参数共享** | Encoder 和 Decoder 参数独立 | 所有层共享一套 Decoder 参数 |
| **长程依赖** | 通过交叉注意力显式对齐源-目标 | 仅靠单向自注意力捕捉上下文 |
| **并行性** | Encoder 完全并行，Decoder 训练时并行但加掩码 | 完全并行训练（因果掩码保证不泄露） |
| **推理方式** | 逐步自回归生成目标词，每步重新计算整个 Decoder | 逐步自回归生成下一个 token，每次追加前缀 |

### 本质理解
- **原始 Transformer**：双塔结构，Encoder 提取源语言的双向上下文表示，Decoder 通过交叉注意力桥接源与目标，然后自回归生成翻译。
- **GPT**：单塔 Decoder，仅用因果自注意力建模文本的左侧历史，通过大量无监督预训练习得通用语言生成能力。

## 6. 附录：常见模型家族速览

| 模型 | 架构 | 特点 |
|------|------|------|
| **GPT 系列** | Decoder‑only | 自回归生成，擅长续写、对话、创意文本 |
| **BERT** | Encoder‑only | 双向理解，擅长分类、抽取、NLU 任务 |
| **T5 / BART** | Encoder‑Decoder | 统一为 Seq2Seq 框架，适用翻译、摘要、问答等 |
| **原版 Transformer** | Encoder‑Decoder | 机器翻译专用，启发后续所有衍生模型 |
